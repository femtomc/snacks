## Preserves -- data model

| Ref                                               | What it contains                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Preserves specification](https://preserves.dev/) | The data model for all Syndicate/Synit/Tiny protocols. Value types: atoms (Boolean, SignedInteger, Double, String, ByteString, Symbol), compounds (Record, Sequence, Set, Dictionary), and Embedded (opaque capability references). Records (`<label field1 field2>`) are the primary protocol structure. Total ordering defined over all values. Three wire formats: text, binary, Syrup. Self-describing (no schema required to parse). Embedded type enables capability references within otherwise pure data. |

## Rhombus and shrubbery notation -- surface syntax

| Ref                                                                                                   | What it contains                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ----------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| [Rhombus documentation](https://docs.racket-lang.org/rhombus/index.html)                              | Racket-based language with shrubbery surface syntax. Tiny's frontend (shrubbery lexer/parser, enforester, operator precedence, macro system) is modeled on Rhombus. Key concepts: indentation-sensitive grouping via `:` and `\|`, enforestation (operator-precedence parsing of shrubbery trees into typed forms), multi-space dispatch (expr, defn, bind spaces).                                                                                                                                    |
| Flatt, [_Binding as Sets of Scopes_](https://doi.org/10.1145/2837614.2837620) (POPL 2016)             | The macro hygiene model. Replaces rename-based hygiene with scope sets. A binding is keyed by name + scope set; a reference resolves to the binding whose scope set is the largest subset of the reference's scope set. Tiny's `syntax.py`, `enforest/hygiene.py`, and `enforest/syntax_meta.py` implement this model directly. Section 3.5 covers use-site scopes that prevent accidental capture.                                                                                                    |
| Flatt et al., _Rhombus: A New Spin on Macros without All the Parentheses_ (OOPSLA 2023)               | Full synthesis paper: shrubbery notation + enforestation + scope-set hygiene + multiple binding spaces + static information protocol. The complete design for macro-extensible conventional-syntax languages. Shrubbery partially groups input (`:` blocks, `\|` alts, delimiters); enforestation (Pratt-style precedence parsing interleaved with macro expansion) handles the rest. Precedence is relational (pairwise, non-transitive), not numeric. Macros receive/return unconsumed syntax tails. |
| Rafkind & Flatt, _Honu: Syntactic Extension for Algebraic Notation through Enforestation_ (GPCE 2012) | Introduces enforestation: Pratt-style precedence parsing interleaved with macro expansion, enabling macros in infix languages. The precursor to Rhombus's design. Tiny's `enforest/enforest.py` implements this algorithm.                                                                                                                                                                                                                                                                             |

## Coordination languages and tuplespaces

Garnock-Jones describes dataspaces as "an adaptation and integration of the fact space model." |
| Callsen & Agha, [_Open Heterogeneous Computing in ActorSpace_](https://doi.org/10.1006/jpdc.1994.1060) (JPDC 21(3), 1994)                                       | Closest actor-tradition precursor to dataspaces. Maps four problems (atomic claims, polling, context decay, flat spaces) to solutions from tuplespace research. Used in the paper's related-work section as the entry point for the SAM-vs-tuplespace argument.                   |

## Verification frameworks

| Ref                                                                                                                                        | What it contains                                                                                                                                                                                                                                                                                                                                                                                   |
| ------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Jung et al., [_Iris from the Ground Up_](https://doi.org/10.1017/S0956796818000151) (JFP 28, 2018)                                         | Higher-order concurrent separation logic. ## Distributed systems and fault tolerance

| De Groen & Garnock-Jones, [_Fine-grained Fault Tolerance in Distributed Training Toolkits using the Syndicated Actor Model_](https://doi.org/10.1145/3774902.3776630) (MIND 2025) | Applies SAM to ML distributed training. | Gamma et al., _Design Patterns_ (Addison-Wesley, 1995)                                                            | Observer pattern. Syndicate's observation model is a language-level reification of Observer with crash cleanup and set semantics.                               |
| Nii, [_Blackboard Systems_](https://doi.org/10.1609/aimag.v7i2.537) (AI Magazine 7(2), 1986)                      | Blackboard architecture: shared workspace with opportunistic knowledge sources. Another precursor to dataspaces.                                                |
| Eugster et al., [_The Many Faces of Publish/Subscribe_](https://doi.org/10.1145/857076.857078) (CSUR 35(2), 2003) | Taxonomy of pub/sub: topic-based, content-based, type-based. Syndicate's observation model subsumes all three via pattern-based assertion matching.             |
| Fagin et al., _Reasoning about Knowledge_ (MIT Press, 2004)                                                       | Epistemic logic: knowledge, belief, common knowledge, distributed knowledge. Formal backdrop for the "three kinds of knowledge" in Garnock-Jones' dissertation. |
| Cardelli & Gordon, _Mobile Ambients_ (TCS 240(1), 2000)                                                           | Scoped, nested computation boundaries. Ambient movement operations. Related to Syndicate's hierarchical dataspaces.                                             |
